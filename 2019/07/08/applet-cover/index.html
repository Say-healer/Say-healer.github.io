小程序中完美分享封面图的解决方案

发布 : 2019-07-08 浏览 :

在小程序中，有很多地方用到分享功能，那分享默认从手机屏幕上从上到下截取一张5:4的图片，来分享，不是很美观，如果是自定义的图片，那么就必须规定好是一张5:4 的图，在很多作品中，不能达到统一，为此来封装个组件，来解决此问题。

1. 创建canvas 画布

2. 设计暴露方法与参数

* properties

根据需求来做的话，可以设计如下参数和方法，multiple根据这个可以调节图片的清晰度，isOpenLoad 可以防止回调没有执行完就分享了。

属性 说明 类型 默认值
customStyle 自定义样式 String -
url 图片路径 String -
multiple 倍数，默认1，宽高为500/400 String -
isOpenLoad 开启 loading Boolean false

event

事件名 说明 返回值
bind:onToSuccessImg 图片绘制成功 e:{detal}
bind:onToErrorImg 图片绘制失败 e:{detal}

3. 获取图片信息

为了保证能加载获取所有的图片，特别是网络图片，可调用wx.getImageInfo，返回临时路径和长宽信息。

4. 渲染画布

为了能够美观，假设最终导出的图以500:400 为基准，那原图分三种情况，

  1. 等于5:4
  2. 大于5:4
  3. 小于5:4

这里，画布先用白色填充，防止出现黑色底边，drawImage方法具体参数如下

参数 描述
img 规定要使用的图像、画布或视频
sx 可选。开始剪切图片的 x 坐标位置
sy 可选。开始剪切图片的 y 坐标位置
sw 可选。被剪切图像的宽度（就是裁剪之前的图片宽度，这里的宽度若小于图片的原宽。则图片多余部分被剪掉；若大于，则会以空白填充）
sh 可选。被剪切图像的高度（就是裁剪之前的图片高度）
dx 在画布上放置图像的 x 坐标位置
dy 在画布上放置图像的 y 坐标位置
dw 可选。要使用的图像的宽度（就是裁剪之后的图片高度，放大或者缩放）
dh 可选。要使用的图像的高度（就是裁剪之后的图片高度，放大或者缩放）

其中，有三种写法，

  • ctx.drawImage(img) // 直接在(0,0) 以图片的宽高画，如果超出canvas 的宽高则隐藏
  • ctx.drawImage(img, dx, dy, dw, dh) // 在画布指定位置上按原图大小绘制指定大小的图(可缩放)
  • drawImage(image, sx, sy, sw, sh, dx, dy, dw, dh) // 所有参数
